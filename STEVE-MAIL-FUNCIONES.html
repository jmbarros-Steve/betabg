🚀 Resumen General

Steve Mail es un módulo de email marketing completo integrado en la plataforma Steve Ads, comparable a Klaviyo o Mailchimp, construido nativamente sobre React + Supabase + Cloud Run.

🎨 Creación

Editor visual drag-and-drop, AI generativa, galería de templates profesionales.

  • Editor GrapeJS Studio SDK
  • 109KB de templates pre-built
  • 14 categorías de plantillas
  • Bloques universales reutilizables
  • Merge tags personalizados
  • Contenido condicional por segmento

👥 Audiencia

Gestión completa de contactos, listas, segmentos y formularios de captura.

  • Sync automático desde Shopify
  • Listas estáticas y segmentos dinámicos
  • Importación CSV
  • Formularios popup/slide-in/inline/full-page
  • Tags y campos personalizados

⚡ Automatización

Flujos automatizados con triggers de Shopify, branching condicional y delays.

  • Carrito abandonado (3 emails)
  • Bienvenida (3 emails)
  • Post-compra (2 emails)
  • Win-back (3 emails)
  • Browse abandonment
  • Cumpleaños

📈 Analítica

Dashboard completo con métricas, heatmaps, atribución de revenue y benchmarks.

  • Open/click/bounce/unsubscribe rates
  • Click heatmaps por enlace
  • Revenue attribution (CLP)
  • Deliverability dashboard
  • Benchmarks de industria
  • Health score 0-100

🤖 Inteligencia AI

Claude AI integrado para generación de contenido, evaluación de calidad y optimización.

  • CRITERIO: evaluación de calidad (18 reglas)
  • ESPEJO: evaluación visual (5 dimensiones)
  • Generación MJML con Claude
  • A/B subject line variants
  • Smart Send Time optimization
  • Product recommendations engine

🔧 Infraestructura

Envío vía Resend, tracking completo, cola con rate limiting, y limpieza automática.

  • Resend API (Amazon SES)
  • Open pixel + click redirect tracking
  • HMAC-signed unsubscribe
  • Send queue con rate limiting
  • Cloud Tasks scheduling
  • List cleanup / sunset automático

⚡ Automatizaciones (Flows)

Motor de flujos automatizados con triggers de Shopify, branching condicional, delays y canvas visual.

🛒 Carrito Abandonado

Trigger: checkouts/create (Shopify webhook)

  • Email 1: Recordatorio suave (1 hora)
  • Email 2: Urgencia (24 horas)
  • Email 3: Descuento / última oportunidad (72 horas)
  • Merge tags: {{ cart_url }}, {{ cart_total }}, {{ cart_items_count }}

👋 Bienvenida

Trigger: customers/create (Shopify webhook) o signup form

  • Email 1: Bienvenida + historia de marca (inmediato)
  • Email 2: Mejores productos (48 horas)
  • Email 3: Incentivo primera compra (7 días)

📦 Post-Compra

Trigger: orders/create (Shopify webhook)

  • Email 1: Agradecimiento + tips de uso
  • Email 2: Solicitud de review + productos complementarios

💔 Win-back

Trigger: Cron job — clientes inactivos

  • Email 1: Te extrañamos
  • Email 2: Novedades y qué hay de nuevo
  • Email 3: Descuento especial de reactivación
  • Cron endpoint: /api/email-flow-cron-winback

🎂 Cumpleaños

Trigger: Cron job — fecha de cumpleaños del suscriptor

  • Email automático con felicitación y oferta
  • Cron endpoint: /api/email-flow-cron-birthday

👁 Browse Abandonment

Trigger: Tracking pixel de navegación

  • Detecta productos visitados sin compra
  • Tracking endpoint: /api/email-flow-track-browse

🤖 Inteligencia Artificial

4 sistemas de AI integrados: generación de contenido, evaluación de calidad (CRITERIO), evaluación visual (ESPEJO) y recomendaciones de producto.

✍ Generación de Contenido AI

Endpoint: /api/generate-steve-mail-content — Claude Sonnet 4.6, 8000 tokens, 4min timeout

  • generate_campaign_html — Email MJML completo con header, greeting personalizado, 5-6 secciones, CTA, footer legal
  • generate_flow_emails — Secuencia multi-email con estrategias por tipo de flow
  • generate_subjects — N variantes de subject + preview con estrategias (urgencia, curiosidad, beneficio, social proof)
  • Carga contexto de marca: buyer persona, catálogo Shopify (20 productos), info del cliente, knowledge base
  • System prompt: "Steve, experto en email marketing" — siempre en español, mobile-first, MJML válido

⚖ CRITERIO — Quality Gate

Endpoint: /api/criterio-email — Motor de reglas determinístico (no LLM)

  • EMAIL SUBJECT (6 reglas): max 50 chars, min 15 chars, preview text 40-130 chars, sin spam words, sin ALL CAPS (<40%), max 2 emojis, distinto a últimos 5
  • EMAIL BODY (8 reglas): CTA presente, link unsubscribe, 50-500 palabras, alt text en imágenes, sin JavaScript, max-width 600px, solo HTTPS, UTM parameters
  • TIMING (3 reglas): hora 8-21, mín 3 días entre envíos, timezone America/Santiago
  • SEGMENT (2 reglas): excluir unsubscribed, mín 100 recipients
  • Output: can_publish (boolean) + score (numérico) + failed_rules
  • Si falla: crea tarea automática en tabla tasks con prioridad critical/high

👁 ESPEJO — Evaluación Visual

Endpoint: /api/espejo — Claude Sonnet 4.6 Vision, 600 tokens

  • MOBILE (1-10): texto legible (min 14px), alt text, CTA visible sin scroll, botones clickeables (min 44px)
  • LAYOUT (1-10): jerarquía visual, espaciado, alineación, max-width ~600px
  • BRAND (1-10): match de colores, logo presente, tono visual coherente
  • CTA (1-10): botón visible y contrastante, texto de acción claro
  • PRODUCTS (1-10): imagen de producto, precios visibles, nombres legibles
  • Umbral: overall ≥ 7 para pasar. Bloquea envío si no pasa.
  • Criterios dinámicos aprendidos de steve_knowledge
  • Si falla: crea tarea automática prioridad alta, squad marketing

🔒 Verificación de Dominio

🔌 API de Dominio

📋 DNS Records Generados

3 registros DNS requeridos para autenticación completa.

  • SPF (TXT): v=spf1 include:amazonses.com ~all
  • DKIM (CNAME): Apunta a infraestructura DKIM de Resend

  • Copy-to-clipboard para cada valor DNS
  • Indicadores de status: verified, pending, failed

🎯 Recomendaciones de Producto

Motor de recomendaciones con 7 estrategias y scoring multi-señal para personalización de emails.

📊 API de Recomendaciones

Endpoint: /api/email-product-recommendations

  • list_products — Catálogo completo para el product picker
  • search_products — Buscar por nombre/handle/type
  • generate — Generar bloque HTML de recomendaciones personalizado
  • preview — Preview de grid de productos
  • clear_cache — Limpiar cache de productos del cliente

🎯 7 Estrategias de Recomendación

Tipos disponibles en RecommendationConfig.type.

  • best_sellers — Por conversiones globales
  • new_arrivals — Por posición en catálogo (recencia)
  • complementary — Via Shopify Recommendations API
  • personalized — Scoring multi-señal (algoritmo principal)
  • not_purchased — Productos no comprados, por popularidad
  • recently_viewed — Basado en historial de clicks
  • abandoned_cart — Recomendaciones basadas en carrito

📊 Scoring Multi-Señal (0-7 pts)

Algoritmo de scoring para recomendaciones personalizadas.

  • +3 pts — Misma categoría que compras previas
  • +2 pts — Producto similar clickeado previamente
  • +1 pt — Producto popular (1+ conversiones globales)
  • +1 pt — Producto nuevo (top 25% del catálogo)
  • Desempate: mayor precio primero

🔄 Cadena de Fallback

Estrategia de degradación cuando faltan datos.

  • 1. Scoring personalizado (si hay historial de compras)
  • 2. Best sellers (si no hay historial, usa conversiones globales)
  • 3. Mayor precio (si no hay datos de conversión)
  • 4. Productos más nuevos (si el subscriber ya compró todo)
