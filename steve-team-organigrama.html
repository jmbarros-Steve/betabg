▼

Qué mantiene

  • Migraciones — cambios al schema sin romper nada
  • steve_sources / swarm_sources — fuentes de información del Brain
  • Integridad — que los datos no se corrompan ni se pierdan

"Tenemos 37 tablas y las 2 más importantes para el Brain tienen 0 filas. Es como tener una Ferrari con el tanque vacío."

Sebastián W5 — Cloud & Crons

▼

Qué mantiene

  • Cloud Run — el servidor donde corre toda la API de Steve
  • 45 crons — todas las tareas automáticas (syncs, análisis, reportes, etc.)
  • 69 edge functions — funciones serverless en Supabase
  • 20 env vars — credenciales que conectan Steve con Meta, Google, Twilio, etc.
  • Health monitoring — OJOS, Sentry, auto-restart

Impacto en el negocio

CriticidadMáxima

Si Cloud Run se cae, NADA funciona. Si un env var se pierde, un módulo entero deja de funcionar. Sebastián es el que mantiene las luces prendidas.

Depende de

Nadie — todos dependen de Sebastián

"Tienes 45 crons y el health-check solo revisa 10 endpoints de 69. Eso es 14% de cobertura. Pregúntale a cualquier cliente si le parece suficiente."

Squad QA — Los que verifican que todo realmente funciona

▼
