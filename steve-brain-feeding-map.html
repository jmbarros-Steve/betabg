1
CONTENT HUNTER
Cada 20 minutos — steve-content-hunter.ts
ROTO Haiku ▼
Problema: La tabla steve_sources tiene 0 rows. El Content Hunter corre cada 20 minutos pero no tiene URLs que escanear. Es como un cartero sin direcciones.

Que hace paso a paso

1.Lee steve_sources WHERE enabled=true, max 5 fuentes, ordenadas por last_checked_at ASC (la mas vieja primero)
2.Para cada fuente, segun source_type:
youtube_channel: Extrae channel_id, baja RSS, parsea 3 videos mas recientes
rss/blog: Baja URL, extrae 3 items mas recientes
website: Encola la URL directamente
3.Para cada contenido nuevo:
- YouTube: Extrae subtitulos del video (captions XML)
- Otros: Intenta Apify primero, si falla hace fetch raw + limpia HTML
- Salta si texto < 200 caracteres
4.Manda a Claude Haiku (max 3,000 tokens): "Extrae reglas accionables de este contenido". Formato: titulo + contenido + categoria. Max 600 chars por regla.
5.Inserta en steve_knowledge con approval_status='pending', orden=2. Las reglas NO se activan hasta que JM las apruebe.
6.Actualiza steve_sources: last_checked_at, last_content_id, total_rules_extracted++

Tablas que toca

TablaLee/EscribeQue hace
steve_sourcesLee + EscribeLee fuentes activas, actualiza last_checked
steve_knowledgeEscribeInserta reglas nuevas (pending)
qa_logEscribeRegistro de ejecucion

Fallas silenciosas (corre pero no hace nada)

Contenido < 200 charsSe salta sin log
YouTube transcript fallatry/catch, salta al siguiente
AI no retorna JSON validoSe salta sin log
Regla < 50 charsSe salta sin log

Para que funcione necesitas

Insertar filas en steve_sources con URLs reales. Ejemplo: HubSpot Blog, Neil Patel, Shopify Blog, Klaviyo Blog, Meta Business Help, canales YouTube de marketing digital. Minimo 10-20 fuentes para que el hunter tenga trabajo.

SWARM RESEARCH
Cada 2 horas — swarm-research.ts
PARCIAL Haiku + o4-mini + Opus ▼
Problema: Solo 16 swarm_runs cuando deberia haber ~360 (12/dia x 30 dias). Las swarm_sources estan vacias (0 rows), asi que Haiku no puede priorizar fuentes. El swarm corre pero sin guia.

Que hace paso a paso (3 cerebros en cascada)

1.DIRECTOR (Haiku) — Genera 10 preguntas de investigacion
- Obtiene el tema del dia segun rotacion tematica (14 temas, AM/PM)
- Carga los 50 titulos mas recientes de steve_knowledge para no repetir
- Carga fuentes preferidas de swarm_sources (0 rows = sin preferencia)
- 7 de 10 preguntas deben referenciar fuentes preferidas, 3 son libres
- Modelo: claude-haiku-4-5-20251001, 1,000 tokens
2.10 McKINSEYS (o4-mini) — Buscan en la web en paralelo
- 10 llamadas simultaneas a api.openai.com/v1/responses
- Cada uno usa web_search_preview tool (busqueda web + analisis)
- Retornan: texto + URLs citadas
- Si una falla, continua con las otras (no bloquea)
3.SENIOR PARTNER (Opus) — Sintetiza insights cruzados
- Recibe los 10 reportes concatenados
- Modelo: claude-opus-4-6, 4,000 tokens
- Cruza hallazgos entre disciplinas (ej: insight de Meta + Klaviyo)
- Retorna max 8 insights con:
  titulo (max 80 chars), contenido (CUANDO/HAZ/PORQUE),
  categorias (1-3), confidence (1-10), sources (URLs)
- Confidence: 8+ = dato duro, 5-7 = tendencia, <5 = hipotesis
4.GUARDAR — Por cada insight, por cada categoria:
- INSERT en steve_knowledge con approval_status='pending'
- Comparte insight_group_id (UUID) para agrupar multi-categoria
- Cuando JM aprueba UNA fila del grupo, se aprueban TODAS las hermanas automaticamente
5.TRACKING — Actualiza swarm_sources: hits++ y last_used_at para las fuentes que aparecieron en la investigacion

STEVE DISCOVERER
Domingos 2am — steve-discoverer.ts
PARCIAL Haiku ▼
Problema: Necesita minimo 20 rows en campaign_metrics para correr. Hoy hay 25 — apenas pasa el umbral pero con muy poca data para descubrir patrones reales.

Que hace paso a paso

1.Lee ultimos 30 dias de: campaign_metrics (spend, clicks, conversions), creative_history (angulo, verdict, score), email_campaigns (subject, opens, clicks)
2.Si < 20 metricas: EXIT SILENCIOSO — retorna "Not enough data"
3.Agrega metricas por dia-de-semana (ej: lunes = $X spend, $Y revenue)
4.Agrega performance de creativos por angulo (total, buenos, malos)
5.Extrae features de emails (largo subject, tiene emoji, tiene numero, open_rate, click_rate)
6.Manda todo a Claude Haiku: "Encuentra correlaciones y patrones inesperados"
7.Inserta en steve_knowledge con titulo [DESCUBIERTO] y approval_status='approved' — se auto-aprueba porque viene de datos reales, no de internet
Diferencia clave: El Discoverer AUTO-APRUEBA sus insights (no necesitan aprobacion de JM) porque se basan en datos medidos, no en busquedas web. Es el unico que hace esto.
4
SALES LEARNING LOOP
Diario 8pm — sales-learning-loop.ts
ROTO Sonnet ▼
Problema: Solo 4 prospectos en wa_prospects y necesita prospectos con stage='converted' o 'lost' que NO hayan sido analizados. Probablemente 0 cumplen ese filtro.

Que hace paso a paso

1.Busca en wa_prospects WHERE stage IN ('converted','lost') AND learning_extracted=false, limit 10
2.Para cada prospecto, carga sus mensajes de wa_messages (channel='prospect', max 100 msgs)
3.Si tiene < 4 mensajes: marca learning_extracted=true y salta (silencioso)
4.Manda conversacion completa a Claude Sonnet (8,000 chars max, 800 tokens). Pregunta: tecnicas que funcionaron, tecnicas que fallaron, punto de quiebre, personalidad, key learning
5.Inserta en steve_knowledge con categoria='sales_learning'. Titulo: "checkmark/X {industria} — {key_learning}"
6.META-PATRONES: Si hay 10+ sales_learnings, Sonnet genera 3-5 meta-patrones cruzados y los guarda como categoria='sales_strategy'

Modelo de AI

Usa Claude Sonnet (no Haiku) porque el analisis de conversaciones requiere mas inteligencia. Es el unico cron de learning que usa Sonnet — mas caro pero mejor calidad.
5
CROSS-CLIENT LEARNING
Mensual (dia 1, 3am) — cross-client-learning.ts
ROTO Haiku ▼
Problema: Necesita 50+ rows en campaign_metrics (hay 25) Y 3+ clientes con spend > $50 (probablemente 0-1). No puede correr.

Que hace paso a paso

1.Lee TODOS los campaign_metrics de los ultimos 30 dias (todos los clientes)
2.Si < 50 registros: EXIT SILENCIOSO
3.Agrupa por connection_id (proxy de cliente). Calcula: spend, CTR, conversion_rate, ROAS
4.Filtra clientes con spend > $50. Si < 3 clientes: EXIT SILENCIOSO
5.Anonimiza: Client_1, Client_2... (nunca revela nombres)
6.Haiku analiza patrones cruzados: "Que tienen en comun los que mejor ROAS tienen?"
7.Upsert en steve_knowledge con titulo [CROSS-CLIENT], auto-aprobado
6
PERFORMANCE EVALUATOR
Diario 10am — performance-evaluator.ts
PARCIAL Haiku ▼
Problema: Necesita creativos medidos (con performance_score) para evaluar. Depende de que Performance Tracker Meta (8am) haya corrido primero. Con 53 creative_history, tiene algo pero poco.

Que hace paso a paso

1.Lee creative_history medidos HOY que no tienen performance_reason
2.Para cada creativo, Haiku analiza: CTR + CPA + ROAS + angulo + tema + benchmarks. Responde en 2 lineas max: POR QUE funciono o fallo.
3.Si verdict = 'malo': crea task de mejora + email al merchant
4.GENERALIZACION DE ANGULOS (el paso clave):
- Lee ultimos 30 dias de creativos con angulo + verdict
- Agrupa por angulo + canal
- Si un grupo tiene 5+ mediciones Y (bueno >=60% O malo >=60%): CREA REGLA
- Ejemplo: "meta: angulo 'descuento' FUNCIONA — score promedio 78/100, 85% tasa exito (12 mediciones)"
5.EFFECTIVENESS SCORING: Para cada regla que fue aplicada en un creativo, actualiza steve_knowledge.effectiveness_score = promedio de los scores de los creativos donde se uso
Este es el loop mas importante: Es el unico que conecta "que angulo usamos" con "como le fue en la realidad" y lo convierte en regla. Sin esto, Steve repite los mismos errores.
7
AUTO-RULE GENERATOR
On-demand — auto-rule-generator.ts
OK Haiku ▼

Que hace paso a paso

1.Recibe un error_detail (ej: "Email sin link de unsubscribe")
2.Lee las primeras 200 criterio_rules activas
3.Haiku decide: ya existe una regla que cubra esto? Si = retorna la existente, no crea nada
4.Si NO existe: genera nueva regla con category, name, check_rule, pass_example, fail_example, severity, weight
5.Inserta en criterio_rules con ID secuencial (R-494, R-495...) y auto=true
Este SI funciona porque es reactivo (no necesita data previa). Cada vez que algun sistema encuentra un error sin regla, este genera una. Es como un sistema inmunologico.
8
STEVE AGENT LOOP
Cada 2 horas — steve-agent-loop.ts
PARCIAL Haiku ▼

Que hace paso a paso (PERCEIVE - REASON - ACT)

PPERCIBIR: Lee alertas recientes de qa_log (ultimas 4h), stats de knowledge (pending vs approved), feedback reciente (positivo vs negativo, 24h), resumen de metricas de clientes (ultimos 3 dias: spend, revenue, ROAS)
RRAZONAR: Haiku recibe el worldState completo y decide 2-3 acciones a tomar. Opciones: search_topic, evaluate_rules, improve_knowledge, alert_client, nothing
AACTUAR:
- search_topic: Busca en YouTube, extrae video IDs, inserta en learning_queue como pending
- improve_knowledge: Busca reglas con quality_score < 40, Haiku las reescribe en formato CUANDO/HAZ/PORQUE, actualiza score a 60
- nothing: Todo bien, no hace nada
EEVALUAR: Revisa steve_commitments por follow-ups vencidos. Si tiene > 30 dias, lo marca como "expired"

Memoria

Cada ejecucion se guarda en steve_episodic_memory (event_type: agent_loop). Esto es la "memoria autobiografica" de Steve — sabe que hizo y cuando. Hoy tiene 13 recuerdos.
9
TRAIN-STEVE (Manual)
Manual — Edge Function
OK Sonnet ▼

Que hace

1.JM o un admin pega contenido (articulo, apunte, estrategia) en la interfaz
2.Claude Sonnet extrae conceptos estructurados: titulo, contenido (lista numerada), categoria, y bugs (errores a evitar con ejemplo bueno/malo)
3.Inserta en steve_knowledge (orden=99, activo=true) y steve_bugs
Esta es la unica fuente 100% manual. Edge function deployada y activa. Funciona cuando JM quiere enseñarle algo especifico a Steve.
10
WOLF SYSTEM (Prospect Intelligence)
3am scrape + 9am send — wolf-night-mode.ts / wolf-morning-send.ts
ROTO Apify + Haiku ▼
Problema: Solo 4 prospectos en wa_prospects. Wolf necesita prospectos con URL de tienda para scrapear. Probablemente ninguno tiene los campos necesarios.

Que hace paso a paso

3amWOLF NIGHT: Para cada prospecto activo con URL de tienda:
- Apify scrapea la tienda (productos, imagenes, precios)
- Compara con scrape anterior: "Subio X producto nuevo", "Cambio de precios"
- Busca anuncios de competencia en competitor_ads
- Guarda findings en wa_prospects.wolf_findings
9amWOLF MORNING: Para prospectos con findings de hoy:
- Haiku genera mensaje WhatsApp natural (4 lineas, tono amigable)
- Envia via Twilio
- Guarda en wa_messages
- Limpia wolf_findings (null)

Que NO alimenta

Wolf NO alimenta steve_knowledge directamente. Alimenta la relacion con prospectos via WhatsApp. El learning viene despues, cuando un prospecto se convierte (o se pierde) y lo procesa Sales Learning Loop (#4).

Mantenimiento: como el cerebro se limpia solo

7 sistemas que mantienen las 487 reglas sanas

Quality Score

Domingos 5am

Calcula score 0-100 para cada regla:

Formato CUANDO/HAZ/PORQUE: +20pts
Tiene numeros/umbrales: +10pts
Usada frecuentemente: +4pts por uso (max 20)
Creada hace <30 dias: +20pts
Tiene ejemplo real: +20pts

Score < 40: Haiku la reescribe automaticamente

Score < 20 + sin usar 60 dias: Se desactiva

Consolidador

Mensual (dia 1, 5am)

Si una categoria tiene > 15 reglas:

Haiku las consolida en 5-8 reglas maestras.

Las viejas se desactivan, las nuevas entran con titulo [CONSOLIDADA] y orden=99.

Umbral: >15 reglas por categoria
Output: 3-10 reglas consolidadas

Deduplicador

Mensual (dia 1, 6am)

Haiku busca duplicados semanticos (no exactos).

Ej: "Usar CTAs claros" y "Los CTAs deben ser directos" = duplicado.

Mantiene la mejor, desactiva las demas, hereda el orden mas alto.

Minimo: 3 reglas por categoria
Minimo total: 5 reglas para correr

Decay (Envejecimiento)

Mensual (dia 1, 4am)

Reglas viejas pierden prioridad o se desactivan. Sin AI — pura logica.

>90 dias sin update + orden >50: baja a orden=50
>180 dias sin update: se desactiva completamente

Rule Calibrator

Domingos 3am

Analiza las 493 reglas de CRITERIO contra datos reales:

Reject rate >80% = "Muy estricta" (si persiste 2+ semanas: auto-disable)
Reject rate <1% con 50+ evals = "Inutil" (candidata a borrar)
Minimo 10 evaluaciones para juzgar

Prompt Evolver

Domingos 3am

Lee feedback de entrenamiento (positivo/negativo) + resultados de JUEZ nocturno.

Genera 3-5 nuevas instrucciones de prompt que se insertan como reglas AUTO-APROBADAS en categoria 'brief'.

Necesita 3+ datos de feedback para correr
Titulo: [AUTO-PROMPT] ...

Propagation Catchup

On-demand

Cuando una regla se aprueba, se propaga a:

CRITERIO = nueva regla de evaluacion

ESPEJO = flag de relevancia visual

JUEZ = nueva pregunta golden para test nocturno

Procesa max 50 reglas aprobadas sin propagar

Steve Chat

🎨

Generate Meta Copy

📧

CRITERIO (493 reglas)

Evalua campañas/emails ANTES de publicar. Bloquea si score < 60.

👁

ESPEJO (Visual)

⚖
