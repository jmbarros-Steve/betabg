▶#173criticalprospectUpdateTags acepta valores arbitrarios — stored XSS + payload stuffingprospect-crm.ts:253

El endpoint valida que tags sea array pero no valida el contenido. Un caller puede enviar [{}, "<script>", null, 12345] directo a DB. Cualquier frontend que renderice tags sin sanitizar es vulnerable a stored XSS. Sin límite de largo del array (podría enviar 10,000 tags).

Fix:
// Validar cada tag: string, no vacío, max 50 chars, max 20 tags const safeTags = tags .filter((t: any) => typeof t === 'string' && t.trim().length > 0) .map((t: string) => t.trim().slice(0, 50).replace(/[<>]/g, '')) .slice(0, 20);
