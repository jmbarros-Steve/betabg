Guía: Conectar Shopify con Custom App Personalizada

Cada cliente tiene su propia Custom App en Shopify. El flujo usa OAuth — el access token se obtiene automáticamente, NUNCA se pide manualmente.

Arquitectura

  • Frontend: React + Vite en Vercel (betabgnuevosupa.vercel.app)
  • Backend API: Hono en Cloud Run (steve-api-850416724643.us-central1.run.app)
  • Base de datos: Supabase (tabla platform_connections)
  • Wizard: src/components/client-portal/ShopifyCustomAppWizard.tsx

Flujo completo (6 pasos)

1 Nombre de la tienda

El usuario ingresa el subdominio de su tienda (ej: mi-tienda para mi-tienda.myshopify.com).

2 Crear Custom App en Shopify Admin

El usuario va a Shopify Admin → Settings → Apps → Develop apps:

  1. Clic en "Desarrollar apps"
  2. Si es primera vez: "Permitir el desarrollo de apps personalizadas"
  3. "Crear una app" → nombre: Steve
  4. Configurar estas URLs:
CampoValor
App URL https://steve-api-850416724643.us-central1.run.app/api/shopify-install
Allowed redirection URL https://steve-api-850416724643.us-central1.run.app/api/shopify-oauth-callback
La App URL DEBE apuntar al endpoint /api/shopify-install del backend, NO al frontend. Esto es necesario para que el link de distribución personalizada redirija correctamente al OAuth.

3 Configurar permisos (API Scopes)

Custom App → "Configure Admin API scopes" → activar estos 11 permisos:

read_products
write_products
read_orders
read_customers
read_checkouts
read_analytics
read_inventory
write_inventory
read_fulfillments
read_discounts
write_discounts

Guardar (Save).

4 Copiar Client ID y Client Secret

Custom App → pestaña "API credentials":

  1. Copiar el Client ID (público, identifica la app)
  2. Copiar el Client Secret (secreto, se usa para OAuth)

Pegarlos en el wizard de Steve. El backend los guarda encriptados en platform_connections.

Endpoint: POST /api/store-shopify-credentials
Body: { clientId, shopDomain, shopifyClientId, shopifyClientSecret }

Esto NO es el Access Token (shpat_...). Son las credenciales de la app para hacer OAuth.

5 Distribución personalizada → Instalar

En la Custom App de Shopify:

  1. Ir a la sección "Distribución"
  2. Clic en "Gestionar distribución personalizada"
  3. Shopify genera un link de instalación único (hash random)
  4. Copiar ese link y pegarlo en el navegador
  5. Shopify muestra pantalla de autorización → clic en "Instalar"

Qué pasa por detrás:

  1. Shopify redirige al App URL (/api/shopify-install) con params shop, hmac, host
  2. shopify-install.ts busca credenciales por shop_domain en DB
  3. Genera nonce, lo guarda en oauth_states, construye URL de autorización
  4. Redirige a https://{shop}/admin/oauth/authorize?client_id=...&scope=...&redirect_uri=...&state=...
  5. Usuario autoriza → Shopify redirige a /api/shopify-oauth-callback con code
  6. Callback intercambia code por access_token
  7. Encripta token → guarda en platform_connections.access_token_encrypted
  8. Redirige al frontend /oauth/shopify/callback?success=true
El endpoint usa window.top.location.href en vez de redirect 302 para romper el iframe si está embebido en Shopify admin.

6 App instalada

El wizard hace polling cada 3s a platform_connections buscando access_token_encrypted IS NOT NULL. Cuando lo encuentra → pantalla de éxito.

Errores comunes

ErrorCausaSolución
"Shopify credentials not found"No se guardaron Client ID/Secret (paso 4)Volver al paso 4
"redirect_uri is not whitelisted"Redirect URL no está en ShopifyAgregar la Redirect URL en la Custom App
"Store URL and Access Token required"OAuth no se completó (paso 5)Completar instalación via distribución personalizada
