email_send_queue NO está rota — está sin usar

La marqué como "ROTA" porque tiene 0 filas. Pero es una feature nueva detrás de un flag, y nadie lo activó todavía.

❌ Lo que dije

"email_send_queue está ROTA: 0 filas, el cron no encuentra nada que procesar, hay un bug en algún lado."

✅ La realidad

La cola es una feature nueva (deployada el 2026-04-08) detrás del flag email_send_settings.use_send_queue. La tabla email_send_settings está vacía: ningún cliente lo tiene en true. Por eso 100% del envío sigue cayendo por la ruta directa de manage-campaigns.ts.

4

Confundí Cloud Tasks con Cloud Scheduler

Son dos servicios distintos de GCP. Cada uno cumple un rol diferente en Steve Mail.

❌ Lo que dije

"Cloud Tasks programa el procesamiento de la cola."

✅ La realidad

Cloud Scheduler (cron) llama al endpoint /api/cron/email-queue-tick cada minuto. Cloud Tasks sí se usa, pero solo para campañas programadas a futuro y para la selección del ganador A/B.

Cloud Tasks Selección de ganador A/B test scheduleAbTestWinner() en manage-campaigns.ts:870-910 con margen de 10min si useSendQueue
