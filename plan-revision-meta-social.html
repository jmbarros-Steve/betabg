S9

Database + Edge Functions + Reporte Final

25 tablas, 6 edge functions, RLS, indexes, y documento final

25
tablas + 6 edge fn

Database — Que reviso

  • RLS policies: cada tabla debe tener policy que filtre por client_id o user_id
  • Tablas sin RLS: las social_* son publicas? verificar que no expongan data privada
  • Indexes: campaign_metrics(client_id, date), competitor_ads(tracking_id), social_posts(created_at)
  • Cascade deletes: borrar client → que pasa con sus campaign_metrics, competitor_tracking?
  • Columnas nullable: campos que nunca deberian ser null pero lo permiten
  • Orphan data: registros en campaign_metrics que apuntan a meta_campaigns que no existen

Edge Functions — Que reviso

  • Duplicacion: sync-campaign-metrics existe en Cloud Run Y en Edge Function — cual se usa?
  • CORS: que las edge functions activas tengan CORS correcto
  • Auth: que validen JWT internamente
  • Dead functions: identificar cuales ya no se usan y marcar para borrar

  • HTML consolidado con todos los hallazgos de las 9 sesiones
  • Lista de bugs encontrados y fixeados (con commits)
  • Lista de tech debt restante (no critico, para despues)
  • Scorecard de salud del modulo: seguridad, performance, UX, data integrity
  • Guardar en Notion + actualizar Supabase
