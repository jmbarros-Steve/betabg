Cloud Run API — Rutas Meta (15 archivos, 7,136 LOC)

Librería Compartida

meta-fetch.ts

Wrapper Graph API v21.0 con:

  • Circuit breaker (previene hammering)
  • Retry exponencial: 2s → 4s → 8s (max 3)
  • Inter-request delay: 200ms (~300 req/min)
  • Timeout: 30s con AbortController
  • Retry-After header support
  • Rate-limit codes: 4, 32, 80004
  • Token en Authorization header (no URL)

Funciones: metaApiFetch(), metaApiJson(), metaApiPaginateAll()

meta-token-refresh.ts

Gestión ciclo de vida tokens:

  • isTokenExpiredError() — code 190, subcodes 463/467
  • refreshMetaToken() — fb_exchange_token grant
  • getValidMetaToken() — refresh proactivo si <7 días
  • handleTokenExpired() — force refresh, marca inactive si falla
  • updateConnectionToken() — encrypta y actualiza DB

OAuth Flow Completo

1. Usuario click "Conectar Meta"
→
2. Genera nonce CSRF → sessionStorage
→
3. Redirect a Meta OAuth (scopes: ads_read, ads_management, business_management, pages_manage_metadata)
→
4. Meta redirect a /oauth/meta/callback?code=...&state=nonce:clientId
5. Frontend valida CSRF
→
6. Envía code a Cloud Run
→
7. Code → short-lived token
→
8. Short → long-lived (~60 días)
9. Fetch businesses + ad accounts (paralelo)
→
10. Encrypt token via RPC
→
11. Save platform_connection
→
12. Auto-sync métricas (fire-and-forget)
