▶#111critical Credit refund TOCTOU: read-then-write pierde créditos entre campañas concurrentes send-campaign.ts:261

El refund de créditos en el finally lee balance, luego escribe balance + unusedCredits. Si dos campañas del mismo cliente terminan simultáneamente, la segunda lectura puede ocurrir antes del primer write — perdiendo un refund entero. Ejemplo: balance=100, campaña A lee 100, campaña B lee 100, A escribe 300, B sobreescribe a 250. Se perdieron 200 créditos.

Problema:
const { data: currentCredits } = await supabase .from('wa_credits').select('balance').eq('client_id', client_id).single(); if (currentCredits) { await supabase.from('wa_credits') .update({ balance: currentCredits.balance + unusedCredits }) .eq('client_id', client_id); }
Fix:
// RPC atómico: UPDATE wa_credits SET balance = balance + p_amount await supabase.rpc('refund_wa_credit', { p_client_id: client_id, p_amount: unusedCredits, p_description: `Refund campaña: ${unusedCredits} no usados` });
