Steve Brain Injection

Plan para que Steve use lo que aprende en TODOS lados, no solo en algunos.

Primero: Que es Steve Brain

Steve Brain es una tabla (steve_knowledge) con ~500 reglas que Steve aprendió de: competencia, feedback de merchants, análisis de performance, y research automático.

Cada regla dice: "CUANDO pasa X, HAZ Y, PORQUE Z".

Ejemplo real: "CUANDO: copy Meta para TOFU. HAZ: usar hook de curiosidad + precio ancla. PORQUE: copies con este patrón tienen 2.4x más CTR en ecommerce Chile."

El Problema

Steve tiene 39 endpoints que usan AI. Pero solo 13 consultan las reglas antes de actuar. Los otros 26 vuelan ciegos.

Que significa "inyectar Brain"

Es agregar 5 líneas de código a cada endpoint. Antes de que la AI genere algo, lee las reglas relevantes de steve_knowledge y se las pasa como contexto. Nada más.

Hoy: Sin Brain

El endpoint llama a Claude con un prompt genérico.

Prompt → Claude → Respuesta

Claude no sabe qué Steve aprendió. Cada respuesta es como si fuera la primera vez.

Después: Con Brain

El endpoint carga reglas relevantes y las inyecta al prompt.

Reglas + Prompt → Claude → Respuesta informada

Claude sabe qué funcionó antes, qué hace la competencia, y qué evitar.

El cambio en código (es esto, en cada archivo):

// ANTES: solo llama a Claude const response = await callClaude(prompt); // DESPUES: carga reglas + llama a Claude const { data: rules } = await supabase .from('steve_knowledge') .select('titulo, contenido') .in('categoria', ['meta_ads', 'anuncios']) // categorías según endpoint .eq('activo', true) .eq('approval_status', 'approved') .limit(15); const knowledgeBlock = rules?.map(r => `- ${r.titulo}: ${r.contenido}`).join('\n'); const response = await callClaude(prompt + '\n\nREGLAS APRENDIDAS:\n' + knowledgeBlock);

Plan de Ejecución

Cada endpoint necesita lo mismo: una función reutilizable que cargue las reglas según la categoría del endpoint. Se crea UNA VEZ en knowledge-loader.ts (que ya existe) y se llama desde cada archivo.

Paso 0
Crear helper reutilizable — Función injectBrain(supabase, categorias[]) en knowledge-loader.ts que retorna el bloque de texto listo para inyectar. Se usa en todos los endpoints. Se escribe una vez.
Paso 1
P1: 5 endpoints de contenido — analyze-brand-research, steve-send-time, steve-bulk-analyze, audit-store, analyze-ad-image. Impacto inmediato en calidad de output para el merchant.
Paso 2
P2: 5 crons de decisiones — auto-postmortem, auto-rule-generator, churn-detector, merchant-upsell, root-cause-analysis. Decisiones automáticas informadas.
Paso 3
P3: 5 endpoints de comunicación — prospect-email-nurture, prospect-followup, wolf-morning-send, onboarding-wa, wa-action-processor. Mejor tono y relevancia.
Paso 4
P4: 5 utilidades — creative-preview, generate-video, parse-email, chonga-support, check-video-status. Fácil, bajo riesgo.
Paso 5
Bonus: 6 write-only — Que cada cron que escribe reglas primero lea las existentes para evitar duplicados y contradicciones.
Paso 6
Build + Deploy + Verificar env vars — Un solo deploy al final de cada paso. Verificar 20/20 env vars.

Resultado Esperado

Hoy

Steve aprende cosas → las guarda → solo las usa cuando el merchant chatea o pide un copy Meta.

Los otros 20 endpoints toman decisiones, generan contenido, y hablan con merchants como si Steve nunca hubiera aprendido nada.

67% del sistema vuela ciego.

Después

Steve aprende cosas → las guarda → TODOS los endpoints las usan automáticamente.

Cada email, cada análisis, cada decisión automática, cada mensaje de WhatsApp lleva el conocimiento acumulado.

95% del sistema informado por Brain.

Para el merchant: Steve es notablemente más inteligente cada semana. Los emails son más relevantes, los análisis más profundos, las recomendaciones más acertadas. Sin que nadie le explique por qué.
