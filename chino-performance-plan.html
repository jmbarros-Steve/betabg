GRUPO A Endpoint Response Time — 12 checks

Mide cuánto tarda cada endpoint crítico de Steve en responder. Usa llamadas reales (POST con body mínimo de prueba) en vez de GET vacío.

Método de medición

  1. Arma request real: método POST/GET, URL del endpoint, body de prueba, headers con Authorization: Bearer service_role_key
  2. Marca t0 = Date.now()
  3. Ejecuta fetch(url, { method, headers, body }) con timeout de 30s
  4. Marca elapsed = Date.now() - t0
  5. Si HTTP status != 2xx → error (endpoint roto)
  6. Si elapsed ≤ max_ms → pass
  7. Si elapsed > max_ms → fail con detalle "Tardó Xms, máximo Yms"

GRUPO B Métricas de Infraestructura — 10 checks

Consultan GCP Monitoring API y Supabase para verificar salud interna de la plataforma. No son endpoints — son métricas del sistema.

B.1 — GCP Cloud Monitoring (6 checks)

Método de medición

  1. Obtener access token GCP: google-auth-library con credenciales del service account de Cloud Run
  2. Llamar monitoring.googleapis.com/v3/projects/steveapp-agency/timeSeries
  3. Filtrar por resource.type = "cloud_run_revision" y resource.labels.service_name = "steve-api"
  4. Leer el valor más reciente del time series
  5. Si valor ≤ threshold → pass
  6. Si valor > threshold → fail
#DescripciónMétrica GCPThresholdCómo mide
89 Memory < 512MB run.googleapis.com/container/memory/utilizations < 0.5 (50% de 1GB) Lee el percentil más reciente del time series. Si > 50% del límite configurado → fail
90 p95 latency < 5s run.googleapis.com/request_latencies < 5,000ms Pide el percentil 95 de las request latencies de los últimos 30 minutos
192 Cold start < 3s run.googleapis.com/request_latencies filtrado por cold start < 3,000ms Filtra requests donde is_cold_start=true, mide p95 de esas latencies
193 Instances < 10 run.googleapis.com/container/instance_count < 10 Lee el valor actual de instancias activas
194 Requests/min < 1000 run.googleapis.com/request_count < 1,000/min Suma request_count del último minuto
200 Error rate < 2% run.googleapis.com/request_count con filtro response_code_class="5xx" < 2% Cuenta 5xx / total requests en últimas 6 horas. Si ratio > 0.02 → fail

Ejemplo: Check #89 — Memory < 512MB
